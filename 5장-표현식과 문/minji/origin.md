## 값

- 값: 표현식(expression)이 평가(evaluate)되어 생성한 결과
- 평가: 식을 해석해서 값을 생성하거나 참조하는 것
- 모든 값은 데이터 타입을 갖고 메모리에 2진수, 즉 비트의 나열로 저장됨
- 값은 다양한 방법으로 생성할 수 있고 그 중 하나가 리터럴을 사용하는 것

## 리터럴

- 리터럴: 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기법
- 리터럴 3 → 자바스크립트 엔진이 이를 평가해 숫자 3을 생성함
- 문자 리터럴, 불리언 리터럴, null 리터럴, 객체 리터럴, 배열 리터럴, undefined 리터럴, 정규표현식 리터럴 등등

## 표현식

- 표현식: 값으로 평가될 수 있는 문(statement)
- 표현식이 평가되면 새로운 값을 생성하거나 기존값을 참조함
- 리터럴도 값으로 평가되므로 표현식
- 표현식과 표현식이 평가된 값은 동치(equivalent)로 값이 위치할 수 있는 곳엔 표현식이 위치할 수 있음

## 문

- 문(statement)은 프로그램을 구성하는 기본 단위이자 최소 실행 단위. 여러 토큰으로 구성됨
- 토큰: 문법적인 의미를 가지며, 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소
  ```jsx
  var sum = 1 + 2; // 문
  // var, sum, =, 1, +, 2, ; 는 토큰
  ```
  - 세미콜론, 마침표 등의 특수 기호도 토큰
- 문은 실행되면 명령이 실행되고 무슨 일인가 일어나게 되어 명령문이라고도 부름
- 선언문, 할당문, 조건문, 반복문 등으로 구분할 수 있음

## 세미콜론과 세미콜론 자동 삽입 기능

- 세미콜론은 문의 종료를 나타내는 토큰
- 문을 끝내야 할 때는 세미콜론을 붙여야 하지만 0개 이상의 문을 중괄호로 묶은 코드 블록 뒤에는 세미콜론을 붙이지 않음 → 코드 블록은 자체 종결성을 가짐
- 문 끝에 붙이는 세미콜론은 자바스크립트 엔진이 소스코드를 해석할 때 문의 끝이라고 예측되는 지점에 세미콜론을 자동으로 붙여주는 세미콜론 자동 삽입 기능이 암묵적으로 수행되므로 생략 가능
  → 제대로 예측하지 못하는 경우가 있어 삽입하는 것을 권장

## 표현식인 문과 표현식이 아닌 문

- 표현식은 문의 일부일 수도있고 그 자체가 문이 될 수도 있음
- 가장 쉬운 구분 방법은 변수에 직접 할당하는 것. 표현식인 문의 경우 변수에 할당이 가능하지만 문의 표현식이 아니면 할당할 수 없음
  ```jsx
  var x; // 선언문의 경우 표현식이 아닌 문
  x = 10; // 할당문의 경우 표현식인 문. var foo = x = 10; 처럼 쓸 수 있음
  ```
- 크롬 개발자 도구에서 표현식이 아닌 문을 실행하면 언제나 undefined를 출력함(완료값)
- 완료 값은 표현식의 평가 결과가 아니므로 변수에 할당할 수도 없고 참조할 수도 없음
