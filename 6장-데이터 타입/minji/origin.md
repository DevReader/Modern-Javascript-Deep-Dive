## 숫자 타입

- C나 자바가 정수, 소수 등 다양한 숫자 타입을 제공하는 것과 달리 한 가지만 제공
- 배정밀도 64비트 부정소수점 형식을 따름 → 정수를 표현하기 위한 데이터 타입이 별도로 존재하지 않음
  - 1 === 1.0 은 true
- 세 가지 특별한 값 표현 가능: Infinity(양의 무한대), -Infinity(음의 무한대), NaN(산술 연산 불가)

## 문자열 타입

- 문자열: 0개 이상의 16비트 유니코드 문자의 집합. 텍스트 데이터를 나타내는데 사용
- 작은따옴표(’’), 큰따옴표(””), 백틱(``)으로 텍스트를 감싸서 표현하고 자바스크립트에서 가장 일반적인 표기법은 작은따옴표
- 문자열을 따옴표로 감싸지 않으면 키워드나 식별자같은 토큰으로 인식함
- 자바스크립트에서 문자열은 원시 타입으로 문자열이 생성되면 그 문자열을 변경할 수 없음

## 템플릿 리터럴

- ES6부터 도입된 새로운 문자열 표기법
- 멀티라인 문자열, 표현식 삽입, 태그드 템플릿 등 편리한 문자열 처리 기능을 제공
- 백틱(``)을 사용해 표현함

### 멀티라인 문자열

- 일반 문자열 내에서는 개행이 허용되지 않아 이스케이프 시퀀스(\n)를 이용해야 함
- 템플릿 리터럴 내에서는 이스케이프 시퀀스 없이도 개행이 허용됨

### 표현식 삽입

- 문자열은 문자열 연산자(+)를 사용한 연결이 가능함
  +) +연산자는 피연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작함
- `${}` 로 표현식을 감싸 표현식을 삽입할 수 있음

## 불리언 타입

- 논리적 참/거짓을 나타내는 true, false

## undefined 타입

- undefined 타입에는 undefined 가 유일함
- 개발자가 의도적으로 변수에 undefined를 할당하는 것은 본래 취지와 어긋나므로 권장하지 않음
  → 변수에 값이 없다는 것을 명시하고 싶을 때는 null 을 할당

## null 타입

- null 타입에는 null 이 유일함
- 변수에 값이 없음을 의도적으로 명시할 때 사용함
- 변수에 null 을 할당하는 것은 이전에 변수가 참조하던 값을 더 이상 참조하지 않겠다는 의미이자 할당되어있던 값에 대한 참조를 명시적으로 제거하는 것
  → 참조하지 않는 메모리에 대해 가비지 콜랙션을 수행할 것
- 함수가 유효한 값을 반환할 수 없는 경우 명시적으로 null을 반환하기도 함

## 심벌 타입

- ES6에서 추가된 7번째 타입
- 다른 값과 중복되지 않는 유일무이한 값으로 충돌할 위험이 없는 객체의 유일한 프로퍼티 키를 만들기 위해 사용
- Symbol 함수를 호출해 생성하고, 생성된 심벌값은 외부에 노출되지 않으며 다른 값과 절대 중복되지 않는 유일무이한 값이 됨

## 객체 타입

- 자바스크립트는 객체 기반의 언어이며, 자바스크립트를 이루고 있는 거의 모든 것이 객체

## 데이터 타입의 필요성

### 데이터 타입에 의한 메모리 공간의 확보와 참조

- 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 결정됨
- ECMAScript 사양은 문자열과 숫자 타입 외 데이터 타입의 크기를 명시적으로 규정하지 않고 있기 때문에 자바스크립트 엔진 제조사의 구현에 따라 다를 수 있음
- 컴파일러 또는 인터프리터는 심벌테이블이라고 부르는 자료 구조를 통해 식별자를 키로 바인딩된 값의 메모리 주소, 데이터 타입, 스코프 등을 관리함

### 데이터 타입에 의한 값의 해석

- 메모리 공간에 저장된 2진수를 데이터 타입에 따라 해석할 수 있음

## 동적 타이핑

### 동적 타입 언어와 정적 타입 언어

- C나 자바 같은 정적 타입 언어는 변수를 선언할 때 변수에 할당할 수 있는 값의 종류, 즉 데이터 타입을 사전에선언해야 함 → 명시적 타입 선언
- 정적 타입 언어는 변수의 타입을 변경할 수 없고, 선언한 타입에 맞는 값만 할당할 수 있음
- 정적 타입 언어는 컴파일 시점에 타입 체크를 수행하고, 타입체크를 통과하지 못했다면 에러를 발생시켜 프로그램의 실행을 방지
- 자바스크립트는 변수를 선언할 때 타입을 선언하지 않고, 어떠한 데이터 값이라도 자유롭게 할당할 수 있음
- 자바스크립트의 변수는 선언이 아닌 할당에 의해 타입이 결정(타입 추론)됨
- 재할당에 의해 변수의 타입은 언제든지 동적으로 변할 수 있음

### 동적 타입 언어와 변수

- 유언성은 높지만 신뢰성은 떨어진다는 단점이 존재
- 변수를 최소한으로 유지해야 함
- 변수의 유효 범위를 최대한 좁게 만들어 변수의 부작용을 억제해야 함
- 전역 변수는 최대한 사용하지 않도록 함
- 변수보다는 상수를 사용해 값의 변경을 억제함
- 변수 이름은 변수의 목적이나 의미를 파악할 수 있도록 네이밍해야 함
