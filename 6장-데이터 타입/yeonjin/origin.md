### 6.2 문자열 타입

문자열 타입은 텍스트 데이터를 나타내는 데 사용

자바스크립트의 문자열은 원시 타입이며, 변경 불가능한 값이다.

### 6.3 템플릿 리터럴

ES6에 도입된 새로운 문자열 표기법 ``

1. 멀티라인 문자열

   일반 문자열 → 이스케이프 시퀀스 사용

   템플릿 리터럴 → 줄바꿈 허용

2. 표현식 삽입

   문자열 연산자 + 를 사용해 연결

   피연산자 중 하나 이상이 문자열인 경우 문자열이 연결 연산자로 동작함

### 6.4 불리언 타입

불리언 타입의 값의 논리적 참, 거짓을 나태는 true와 false

### 6.5 undefined 타입

undefined 타입의 값은 undefined가 유일함

var 키워드로 선언한 변수는 암묵적으로 undefined로 초기화됨

변수를 선언한 이후 값을 할당하지 않은 변수를 참조하면 undefined가 반환됨

### 6.6 null 타입

null 타입 값은 null이 유일함

변수에 값이 없다는 것을 의도적으로 명시할 때 사용한다.

### 6.7 심벌 타입

ES6에 추가된 7번째 타입으로, 변경 불가능한 원시 타입임.

심벌 값은 다른 값과 중복되지 않는 유일무이한 값임.

이름이 충돌할 위험이 없는 객체의 유일한 프로퍼티 키를 만들기 위해 사용함.

```jsx
var key = Symbol("key");
console.log(typeof key); // symbol

var obj = {};

obj[key] = "value";
console.log(obj[key]); //value
```

### 6.9 데이터 타입의 필요성

1. 데이터 타입에 의한 메모리 공간의 확보와 참조

   메모리에 값을 저장하려면 먼저 확보해야 할 메모리 공간의 크기를 결정해야 함

   (숫자의 크기 = **자료형 정의 + 언어 스펙 + CPU/OS 아키텍처**에 따라 달라짐)

2. 데이터 타입에 의한 값의 해석
   - 값을 저장할 때 확보해야 하는 메모리 공간의 크기를 결정하기 위해
   - 값을 참조할 때 한 번에 읽어 들여야 할 메모리 공간의 크기를 결정하기 위해
   - 메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해

### 6.10 동적 타이핑

1. 동적 타입 언어와 정적 타입 언어

   자바스크립트는 변수를 선언할 때 타입을 선언하지않음. 선언이 아닌 할당에 의해 타입이 결정(추론)된다. 재할당에 의해 변수의 타입은 동적으로 변할 수 있다 (=동적 타이핑)

2. 동적 타입 언어와 변수

   타입이 언제든 변경될 수 있기 때문에 프로그램에서 변화하는 변수 값을 추적하기 어려울 수 있다. → 유연성은 높지만 신뢰성은 떨어진다.
