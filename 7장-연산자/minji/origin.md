## 산술 연산자

- 산술 연산자는 피연산자를 대상으로 수학적 계산을 수행해 새로운 숫자 값을 생성.
- 산술 연산이 불가능한 경우는 NaN 반환

### **이항 산술 연산자**

- 이항 산술 연산자는 2개의 피연산자를 산술 연산하여 숫자 값 생성
- 어떤 산술 연산을 해도 피연산자의 값이 바뀌지는 않고 언제나 새로운 값을 만들어 냄
- 덧셈 연산자(+), 뺄셈 연산자(-), 곱셈 연산자(\*), 나눗셈 연산자(/), 나머지 연산자(%)

### 단항 산술 연산자

- 단항 산술 연산자는 1개의 피연산자를 산술 연산하여 숫자 값 생성
- 주의할 점은 증가/감소(++/--) 연산자는 피연산자의 값을 변경하는 부수 효과가 있음 → 증가/감소 연산을 하면 피연산자의 값을 변경하는 암묵적 할당이 이루어짐
  - 전위 증가/감소 연산자는 먼저 피연산자의 값을 증가/감소시킨 후, 다른 연산을 수행
  - 후위 증가/감소 연산자는 먼저 다른 연산을 수행한 후, 피연산자의 값을 증가/감소 수행
- 한편, +와 - 단항 연산자는 숫자 타입이 아닌 피연산자에 사용하면 피연산자를 숫자 타입으로 변환하여 반환
  이때 피연산자를 직접 변경하지는 않고 숫자 타입으로 변환한 값을 생성해 반환하므로 부수 효과는 없음

### 문자열 연결 연산자

- +연산자는 피연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작하고 그 외에는 산술 연산자로 동작
- 개발자의 의도와는 상관없이 자바스크립트 엔진에 의해 암묵적으로 타입이 자동 변환되기도 하는데, 이를 암묵적 타입 변환 또는 타입 강제 변환이라 함
  - `1 + true;` true 를 숫자 타입 1로 강제 변환해 연산을 수행해서 2가 됨
  - `+undefined` , `+’Hello’` : NaN

## **할당 연산자**

- 할당 연산자: 피연산자의 평가 결과를 좌항에 있는 변수에 할당 → 변수 값 변함(부수 효과 O)
- 종류: =, +=, -=, \*=, /=, %=
- 할당문은 문이므로 여러 변수에 동일한 값을 연쇄적으로 할당할 수 있음
  ```jsx
  var a, b, c;
  a = b = c = 0; // 오른쪽 -> 왼쪽으로 진행. c = 0 -> b = 0 -> a = 0
  ```

## **비교 연산자**

- 비교 연산자: 좌항과 우항의 피연산자를 비교한 다음 그 결과를 불리언 값으로 반환

### **동등/일치 비교 연산자**

- 동등 비교: 느슨한 비교. (==, !=)
  좌항과 우항의 피연산자를 비교하기 전 암묵적 타입 변환을 통해 타입을 일치시킨 후 같은 값인지 비교함
  → 예측하기 어려운 결과를 만들어내므로 일치 비교 연산자 사용하는 것이 좋음
- 일치 비교: 엄격한 비교(===, !==)
  - 좌항과 우항의 피연산자가 타입도 같고 값도 같은 경우에 한하여 true 를 반환
  - NaN은 자신과 일치하지 않는 유일한 값이므로(`NaN === NaN` 은 false) NaN 인지 조사하려면 `Number.isNaN` 을 사용함
  - `0 === -0` 은 true ES6에서 도입된 `Object.is` 를 쓰면 둘을 구분 가능

### **대소 관계 비교 연산자**

- 대소 관계 비교 연산자: 피연산자의 크기를 비교하여 불리언 값을 반환

## **삼항 조건 연산자**

- 삼항 조건 연산자: 조건식의 평가 결과에 따라 반환할 값을 결정 → 부수 효과 X
  ```jsx
  조건식 ? 조건식이 true일 때 반환할 값 : 조건식이 false일 때 반환할 값
  ```
- 조건식의 평가 결과가 불리언 값이 아니면 불리언 값으로 암묵적 타입 변환
- 삼항 조건 연산자 표현식은 값처럼 사용할 수 있지만 if - else 문은 값처럼 사용 불가

## **논리 연산자**

- 논리 연산자: 좌항과 우항의 피연산자(부정 논리 연산의 경우 우항의 피연산자)를 논리 연산 함
- ||, &&, !(부정논리연산자)
- 부정 논리 연산자는 언제나 불리언 값을 반환하고 피연산자가 불리언 값이 아니면 암묵적 타입 변환함
- 논리합 또는 논리곱 연산자의 평가 결과는 불리언 값이 아닐 수 있고 2개의 피연산자 중 어느 한쪽으로 평가됨

## **쉼표 연산자**

- 왼쪽 피연산자부터 차례대로 피연산자를 평가하고 마지막 피연산자의 평가가 끝나면 마지막 피연산자의 평가 결과를 반환함
  ```jsx
  var x, y, z;
  (x = 1), (y = 2), (z = 3); // 3
  ```

## **그룹 연산자**

- 그룹 연산자: 자신의 피연산자인 표현식을 가장 먼저 평가(`()`)

## **typeof 연산자**

- typeof 연산자는 피연산자의 데이터 타입을 문자열로 반환
- 7가지 문자열 "string", "number", "boolean", "undefined", "symbol", "object", "function" 중 하나를 반환
- null을 반환하는 경우 없음(null 값은 object 로 연산)
- 선언하지 않은 식별자를 연산할 땐 ReferenceError 가 아닌 undefined 반환

## **지수 연산자**

- 지수 연산자: ES7에서 도입. 좌항의 피연산자를 밑(base)으로, 우항의 피연산자를 지수(exponent)로 거듭 제곱하여 숫자 값을 반환
- 이전에는 `Math.pow` 메서드 사용
- 다른 산술 연산자와 마찬가지로 할당 연산자와 함께 사용 가능

| in         | 프로퍼티 존재 확인                                          | 19.13.1절 |

## **연산자의 부수 효과**

- 대부분의 연산자는 다른 코드에 영향을 주지 않지만 일부 연산자는 다른 코드에 영향을 주는 부수 효과 존재
- 부수 효과가 있는 연산자: 할당 연산자(=), 증가/감소 연산자(++/--), delete 연산자

## **연산자 우선순위**

- 연산자 우선순위: 여러 개의 연산자로 이루어진 문이 실행될 때 연산자가 실행되는 순서
- 기억하려하기 보다는 그룹 연산자 이용해서 우선순위를 명시적으로 조절하는 것을 권장

## **연산자 결합 순서**

- 연산자 결합 순서: 좌항과 우항 중 어느 쪽부터 평가를 수행할 것인지를 나타내는 순서

| **결합 순서** | **연산자**                                                                                   |
| ------------- | -------------------------------------------------------------------------------------------- | --- | ----------------------------------- |
| 좌항 → 우항   | +, -, /, %, <, <=, >, >=, &&,                                                                |     | , ., [], (), ??, ?., in, instanceof |
| 우항 → 좌항   | ++, --, 할당 연산자(=, +=, -=, ...), !x, +x, -x, ++x, --x, typeof, delete, ? ... : ..., \*\* |
